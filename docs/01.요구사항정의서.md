# 요구사항 정의서

## 프로젝트 개요

| 항목        | 내용                                               |
| ----------- | -------------------------------------------------- |
| 프로젝트명  | My Dev Blog                                        |
| 목적        | 개발자가 학습 내용을 기록하고 공유하는 개인 블로그 |
| 대상 사용자 | 개발 학습자, 주니어 개발자                         |
| 개발 기간   | 9일 (Day 4 ~ Day 9)                                |

## 기능적 요구사항

### 사용자 인증 (Authentication)

| ID       | 요구사항                                        | 우선순위 |
| -------- | ----------------------------------------------- | -------- |
| AUTH-001 | 사용자는 이메일과 비밀번호로 회원가입할 수 있다 | 필수     |
| AUTH-002 | 사용자는 이메일과 비밀번호로 로그인할 수 있다   | 필수     |
| AUTH-003 | 사용자는 Google 계정으로 로그인할 수 있다       | 필수     |
| AUTH-004 | 사용자는 로그아웃할 수 있다                     | 필수     |
| AUTH-005 | 시스템은 로그인한 사용자의 상태를 유지한다      | 필수     |

### 게시글 관리 (Post)

| ID       | 요구사항                                        | 우선순위 |
| -------- | ----------------------------------------------- | -------- |
| POST-001 | 로그인한 사용자는 새 게시글을 작성할 수 있다    | 필수     |
| POST-002 | 시스템은 게시글 목록을 최신순으로 표시한다      | 필수     |
| POST-003 | 사용자는 게시글 상세 내용을 조회할 수 있다      | 필수     |
| POST-004 | 작성자는 자신의 게시글을 수정할 수 있다         | 필수     |
| POST-005 | 작성자는 자신의 게시글을 삭제할 수 있다         | 필수     |
| POST-006 | 사용자는 게시글을 카테고리별로 필터링할 수 있다 | 선택     |
| POST-007 | 사용자는 게시글을 검색할 수 있다                | 선택     |
| POST-008 | 로그인한 사용자는 게시글에 이미지를 첨부할 수 있다 | 필수     |
| POST-009 | 게시글 삭제 시 관련된 댓글과 이미지가 함께 삭제된다 | 필수     |

### 댓글 관리 (Comment)

| ID       | 요구사항                                        | 우선순위 |
| -------- | ----------------------------------------------- | -------- |
| CMT-001  | 로그인한 사용자는 게시글에 댓글을 작성할 수 있다 | 필수     |
| CMT-002  | 비로그인 사용자는 댓글 작성 시 로그인 페이지로 이동한다 | 필수     |
| CMT-003  | 작성자는 자신의 댓글을 삭제할 수 있다           | 필수     |
| CMT-004  | 사용자는 다른 사람의 댓글을 삭제할 수 없다      | 필수     |
| CMT-005  | 사용자는 게시글의 댓글 목록을 조회할 수 있다    | 필수     |

### 좋아요 기능 (Like)

| ID       | 요구사항                                        | 우선순위 |
| -------- | ----------------------------------------------- | -------- |
| LIKE-001 | 로그인한 사용자는 게시글에 좋아요를 누를 수 있다 | 필수     |
| LIKE-002 | 한 포스트당 한 사용자는 한 번만 좋아요를 누를 수 있다 | 필수     |
| LIKE-003 | 이미 좋아요를 누른 경우 다시 클릭하면 좋아요가 취소된다 | 필수     |
| LIKE-004 | 좋아요 버튼 옆에 총 좋아요 수가 표시된다        | 필수     |
| LIKE-005 | 좋아요 컴포넌트는 게시글 목록과 상세 페이지에 모두 표시된다 | 필수     |
| LIKE-006 | 비로그인 사용자는 좋아요 클릭 시 로그인 페이지로 이동한다 | 필수     |

### 사용자 경험 (UX)

| ID     | 요구사항                                              | 우선순위 |
| ------ | ----------------------------------------------------- | -------- |
| UX-001 | 비로그인 사용자도 게시글 목록과 상세를 조회할 수 있다 | 필수     |
| UX-002 | 로그인이 필요한 기능 접근 시 로그인 페이지로 이동한다 | 필수     |
| UX-003 | 모바일 환경에서도 정상적으로 표시된다                 | 필수     |

## 비기능적 요구사항
